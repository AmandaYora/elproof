# Product Requirements — ElProof

Product purpose, users, scope, modules, business flow.

ElProof is a SaaS for Wedding Organizer (WO) businesses: a WO Console for tenant staff to run
projects/vendors/clients, a Client Portal for the bride/groom/family to track their wedding, and a
Platform Console for ElProof's own team to run the SaaS business itself (tenants, plans, billing).

Full detail — users, per-surface feature list, explicit out-of-scope items — lives in
[`knowledge/PROJECT_BRIEF.md`](../knowledge/PROJECT_BRIEF.md) and
[`knowledge/PRODUCT_REQUIREMENTS.md`](../knowledge/PRODUCT_REQUIREMENTS.md) (not duplicated here).

The phased plan to build the backend and integrate it with the existing frontend is tracked in
[`PLAN.md`](../PLAN.md) at the repo root.
